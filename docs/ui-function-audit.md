# UI 与功能实现对照检查

检查日期：2026-07-08

依据：

- `需求以及资料/功能实现.md`
- `docs/frontend-conventions.md`
- 当前 `app/ui/`、`app/services/`、`app/device/`、`app/api/` 实现

说明：本文件记录当前软件页面可用性、与功能范围的符合程度，以及后续 UI 优化优先级。不修改需求资料原文。

## 总体结论

当前项目不是空壳，主要业务页面和服务已经覆盖：登录授权、账号权限、实时监控、地图监控、设备卡片、曲线、记录、设备配置、设备调试、备份恢复、本地 API、维护提醒、报警联动、大屏展示均已有实现或入口。

主要问题集中在三类：

1. 页面视觉一致性仍需统一，尤其是 Tab、表格、过滤区、页面头部和二级导航。
2. 部分功能已在服务层实现，但 UI 暴露不足或入口刚补齐，例如账号管理。
3. 面向现场值守人员的流程还可继续简化，例如默认账号记忆、减少重复提示、减少需要理解技术名词的输入。

## 已发现功能差距

| 优先级 | 差距 | 影响 |
| --- | --- | --- |
| P0 | 打包交付流程和免命令行初始化仍需最终验证 | 客户现场无法接受命令行操作 |
| P0 | 授权注册机已做 UI，但还需和主程序打包流程联动 | 甲方交付授权码需要稳定工具 |
| P1 | 曲线筛选仍偏技术 ID 输入 | 普通客户难以使用 |
| P1 | 系统配置内表单字段较多，部分字段需要更清晰的帮助文案 | 客户配置探测器时容易填错 |
| P1 | 账号管理 UI 没有直接“删除账号”按钮，偏启用/禁用 | 功能实现要求“删除”，服务层有删除能力但 UI 未暴露 |
| P2 | 多页面页面头部样式不完全统一 | 专业感不足 |
| P2 | 真实设备联动协议未确认 | 只能模拟联动，不能真实控制外设 |

## 本轮已处理

- 登录窗口支持记住上次成功登录账号，仅保存用户名，不保存密码。
- 账号管理入口已加入主导航，并统一页面标题。
- 主界面切页普通提示已隐藏，仅保留异常/无权限提示。
- 底部状态栏已压缩为单行。
- 全局优化了 Tab、表格、滚动区样式，减少 Qt 默认控件风格割裂。
- 系统配置页二级分类已从左侧竖排改为顶部横向分段按钮。

## 下一轮建议

1. 将曲线和记录筛选中的“探测器 ID 输入”升级为控制器/探测器选择控件。
2. 给设备调试页增加“复制调试结果”。
3. 给账号管理页增加“重置密码”和“删除账号”显式操作，并保留唯一管理员保护。
4. 优化系统配置页字段帮助文案，尤其是协议地址、寄存器序号、报警阈值、传感器寿命和校验周期。
5. 做一次打包后完整流程验证：首次启动、默认管理员、强制改密、授权、登录、注册机生成授权码。
