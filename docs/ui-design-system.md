# 堃联工业桌面 UI 设计系统

本项目是给现场值守、设备维护和甲方管理人员使用的 Windows 桌面软件。UI 的目标不是炫技，而是让用户在低培训、长时间值守、偶发紧急事件下仍然看得懂、点得准、找得到。

## 设计身份

后续页面默认按这三个身份共同审视：

- 工业安防监控软件产品设计师：优先处理报警、状态、误操作和现场可读性。
- B 端桌面工具信息架构师：页面要像工作台，不像宣传页，也不像堆卡片的网页。
- Qt 前端工程师：尊重 Qt Widgets 的布局机制，避免靠无限拉伸输入框假装规整。

## 页面结构

标准业务页采用四层结构：

1. 页头：标题、简短说明、当前页主要操作。页头不做厚卡片，高度控制在 56-72px。
2. 二级导航或筛选：只在需要切换数据域或过滤数据时出现，视觉上比页头弱。
3. 主数据区：表格、列表、监控卡片或地图，是页面主体。
4. 编辑/详情区：靠近主数据区，短字段短、长字段长，不因窗口变宽而分裂到两边。

## 布局规则

- 页面外边距统一 12-16px，不做大面积留白。
- 表格优先占满横向空间，表单不占满横向空间。
- 表单使用固定节奏：标签 82px，短输入 120-160px，中输入 220-280px。
- 表单字段组靠左排列；右侧留白可以存在，但不能把字段推散。
- 页头操作按钮放在右侧；同类动作顺序为：新增、保存、删除、刷新、导入、导出。
- 不把页面 section 做成一层又一层卡片。只有表格容器、表单容器、弹窗、重复项可以有边框。

## 颜色和状态

- 主色使用稳定工业蓝，不用大面积高饱和蓝。
- 背景以浅灰蓝和白色为主，边框要清楚但不厚重。
- 危险状态只用于删除、高报、故障，不用于普通强调。
- 选中行必须明显；禁用控件要明显置灰。

## 控件规则

- 表格行高 30-34px，表头浅灰蓝，选中行为蓝底白字。
- 输入框默认高度 30-32px；不要让短数字输入框占满一整列。
- 主按钮最多 1 个；同一区域不要多个蓝色按钮。
- 页面内提示默认短暂出现，除报警和安全风险外不常驻。
- 时间显示给用户看本地格式：`YYYY-MM-DD HH:mm`，不显示 ISO 字符串。

## 系统配置页规则

- 顶部分类导航是紧凑 segmented control，不横向铺满成大按钮墙。
- 表格在上方作为当前配置清单。
- 表单在下方作为编辑区，字段组靠左，右侧空白保留。
- 气体类型的单位、量程、默认阈值必须能被探测器配置自动继承。

## 账号管理页规则

- 页头紧凑展示标题、说明和操作。
- 筛选条不做大卡片，保持一行扫描。
- 用户表格展示业务语言：管理员/操作员、启用/禁用、本地时间。
- 权限不足提示不占用主体表格空间。
