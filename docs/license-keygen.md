# 授权码生成工具

本工具用于供应商或甲方根据客户机器码生成授权码或 `.lic` 文件。它不连接客户数据库，也不修改主程序运行数据。

## 当前流程

1. 客户启动主程序，进入“软件授权”窗口。
2. 客户点击“复制机器码”，把完整机器码发给供应商或甲方。
3. 供应商或甲方打开注册机 UI，生成授权码或 `.lic` 文件。
4. 客户在“软件授权”窗口粘贴授权码，或导入 `.lic` 文件完成激活。

## 打开注册机 UI

PowerShell：

```powershell
.\scripts\license-keygen.ps1
```

注册机窗口中填写：

- 机器码：客户授权窗口复制出的完整机器码。
- 授权期限：默认永久授权；取消勾选后可填写到期日期。
- 客户名称/备注：仅用于内部核对，会写入授权载荷。

点击“生成授权码”后，可复制授权码或保存 `.lic` 文件。记录会自动进入“客户列表”页。

## 客户列表

“客户列表”页用于甲方保存授权台账。每一行包含：

- 客户名称
- 机器码
- 授权类型
- 到期时间
- 备注
- 生成时间

同一机器码再次生成授权会更新原记录。选中记录后可以复制授权码，也可以载入到生成页重新生成。

## 命令行生成授权码

注册机 UI 是推荐方式；命令行只用于内部测试或自动化。

PowerShell 示例：

```powershell
python tools\license_keygen.py --machine-code "<客户机器码>" --expires-at 2027-12-31 --customer-name "客户名称" --output customer.lic
```

`--expires-at YYYY-MM-DD` 会按北京时间当天 `23:59:59+08:00` 过期。不传 `--expires-at` 表示永久授权。

## 客户侧规则

客户侧不需要命令行，也不需要设置环境变量。客户只需要：

1. 在授权窗口复制机器码。
2. 把机器码发给甲方或供应商。
3. 收到授权码后粘贴，或导入 `.lic` 文件。

当前版本内置产品授权校验密钥，后续正式商用建议升级为“私钥生成、公钥校验”的非对称签名方案。
